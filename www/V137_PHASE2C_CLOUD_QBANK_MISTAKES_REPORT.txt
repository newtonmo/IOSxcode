NovaMed v137 - Phase 2C Cloud QBank + Mistakes

What changed:
1. QBank and timed exam question attempts now write to Supabase table qbank_attempts.
2. Wrong answers and My Mistakes now write/update Supabase table question_mistakes.
3. On student sign-in, QBank attempts and mistakes are restored from Supabase into the app state.
4. My Mistakes actions (mastered/reviewed/recovered/personal note/review outcome) sync back to question_mistakes.
5. Browser cache version updated to v137-phase2c-cloud-qbank-mistakes.

Tables used:
- qbank_attempts
- question_mistakes

Test plan:
1. Sign in as a student.
2. Solve a correct QBank MCQ. Check qbank_attempts has a new row.
3. Solve a wrong MCQ. Check qbank_attempts and question_mistakes.
4. Open My Mistakes, add a personal note, mark reviewed/mastered, and confirm question_mistakes updates.
5. Sign out, sign in again, and confirm attempts/analytics/mistakes restore from Supabase.

Notes:
- This phase does not change content importing or admin QBank JSON workflow.
- Supabase Auth session may still exist in browser; academic progress is saved to Supabase tables.
