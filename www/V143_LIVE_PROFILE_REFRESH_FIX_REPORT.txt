NovaMed v143 - Live profile refresh + responsive workspace fix

Built on v142 public leaderboard safe restore.

Fixes:
1. Profile/Street counters update from in-memory state immediately after XP, mistakes, bookmarks, flashcards, and student_state saves; no refresh required.
2. Current signed-in student is patched into the leaderboard cache immediately, then synced to Supabase in the background.
3. Profile workspace buttons (My Mistakes, Analytics, Flashcards, Bookmarks) now have an additional capture-phase guard so they remain clickable even after large lists are rendered.
4. Hidden modal/backdrop cannot intercept Profile clicks.
5. Bookmarks panel uses event delegation instead of attaching a separate listener to every saved item after each render, improving responsiveness with many bookmarks.
6. Mistake-note typing does not force re-render of the active textarea, preventing focus loss.

No new SQL is required if v142 SQL already ran successfully.
