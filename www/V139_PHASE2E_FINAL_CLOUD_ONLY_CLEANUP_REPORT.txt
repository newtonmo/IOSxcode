NovaMed v139 - Phase 2E Final Cloud-only cleanup

Goal:
- Keep student progress in Supabase tables, not localStorage.
- Keep only Supabase Auth/browser session behavior as the expected browser-side persistence.

Completed in this build:
1. Student UI progress no longer writes to novamed-ui-progress-v1.
   - It is cached in memory during the session and pushed to student_state.

2. Student saveState no longer writes content/progress snapshots to localStorage.
   - Student progress is saved through saveStudentProgress -> student_state/student_public.
   - Admin content publishing still uses the existing content cloud workflow.

3. Exam targets and study route planner no longer write student values to localStorage.
   - They are restored into in-memory caches.
   - They are included in the student profile snapshot and saved to student_state.

4. Home widget settings no longer write student values to localStorage.
   - They are cached in memory and included in student_state.

5. Imported QBank packages no longer persist in novamed-imported-qbank-v87.
   - During import they are held in memory.
   - Admin imported MCQs are pushed into the app QBank/content save workflow.

6. Legacy local student keys are cleaned after boot/sign-in:
   - novamed-ui-progress-v1
   - novamed-feature-layer-v79
   - novamed-imported-qbank-v87
   - novamed-home-widget-settings-v117
   - novamed-live-timed-exam-v1
   - novamed-smart-study-plan-v85
   - novamed-exam-targets-v1:* 
   - novamed-study-route-v1:*

7. Street/Streak remains real-students-only using student_public.

Expected remaining browser storage:
- Supabase Auth may keep a session token so the student stays signed in.
- Admin/local-owner setup and non-student fallback UI may still use browser storage.
- Student learning progress should now be stored in Supabase tables.

Recommended test:
1. Upload this build to Netlify.
2. Sign in as a real student.
3. Change theme/path/widget settings, create study plan, start live exam, watch video, add note, solve QBank, bookmark, review flashcards.
4. Check Supabase tables: student_state, student_public, live_exams, study_plans, video_progress, student_notes, qbank_attempts, question_mistakes, flashcard_progress, student_bookmarks, student_search_history.
5. Refresh and sign in on another browser to verify restoration from cloud.
