NovaMed simple login + Supabase profile sync notes

Supabase Auth / Email OTP remains disabled.
Students use only:
- Student name
- Simple code

This build requires Supabase for student accounts because the goal is multi-device restore.
It will not silently create a local-only student account if Supabase URL/anon key/table/RLS are not ready.

If a second device says "account created", the old build was probably running or supabase-config.js was empty/cached.
This version fixes that by:
- fetching student_profiles before account creation,
- verifying that upserted profiles are readable,
- showing the real Supabase error,
- avoiding service-worker caching of supabase-config.js.

Run SUPABASE_SETUP.sql and embed Project URL + anon public key in supabase-config.js before publishing.
